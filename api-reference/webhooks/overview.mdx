title: Webhooks Overview
description: Receive real-time updates for transaction and account status changes
---

Teel sends webhook notifications to your configured endpoints when events occur, such as transaction status updates, KYC completions, and account changes.

## Supported Providers

| Provider | Endpoint | Signature Verification |
|----------|----------|----------------------|
| Transfi | `POST /api/transfi/webhook` | HMAC-SHA256 |
| PDAX | `POST /api/pdax/webhook` | None |
| Conduit | `POST /api/conduit/webhook` | HMAC-SHA256 |
| Airwallex | `POST /api/airwallex/webhook` | HMAC-SHA256 |
| Borderless | `POST /api/borderless/webhook` | None |
| OnrampMoney | `POST /api/onrampmoney/webhook` | Custom signature |
| Luno | `POST /api/luno/webhook` | None |

## Common Behavior

All webhook handlers follow the same pattern:

1. Verify signature (where applicable)
2. Parse the JSON request body
3. Look up the transaction in the database
4. Update transaction status
5. Broadcast status update via WebSocket
6. Auto-trigger offramp when onramp completes (where applicable)

## Responding to Webhooks

Return a `200 OK` response to acknowledge receipt. If a non-2xx response is returned, the provider may retry the webhook.
